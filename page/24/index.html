Local Storage에 저장하기

March 19, 2019 Javascript

Local Storage

내가 만든 페이지가 어떤 데이터를 기억하고,
새로고침을 해도 데이터를 잃어버리지 않게 하고 싶다면
local storage 에 저장하는 방법이 있다.

( + 세션 스토리지와 로컬 스토리지
세션 스토리지는 로컬 스토리지와 비슷하다.
다른 점은, 로컬 스토리지에 저장된 데이터는 새로고침을 하거나 창을 닫았다 열어도
데이터를 직접 지우지 않는 이상 그대로 남아있다.
세션 스토리지에 저장이 되면 세션이 끝난 후에, 즉 브라우저를 닫는 경우에 데이터가 사라진다. )

기본 문법은 위와 같고 필요한 기능에 따라 적절히 수정하여 쓰면 된다.
오늘은 로컬스토리지에 데이터를 저장하고 불러오는 두 가지 방법을 배웠다.
serialize(); 를 이용하는 방법과
dictionary 를 이용하는 방법이다.

serialize();

폼 태그 안의 데이터를 문자열로 합쳐주는 메소드이다.

save 버튼을 누르면 로컬 스토리지에 content 라는 키가 생기고
serialize 를 썼기 때문에 value 는
content1=hela&content2=thor&content3=loki
이런 식으로 생기게 된다.
load 버튼을 누르면 다시 데이터가 화면에 보이게 했는데,
그때 저장된 값을 불러와 split(); 을 이용해서 & 를 기준으로 나눠주면
콘솔에 띄웠을 때

이렇게 나뉘어서 배열이 생긴다.
그 안에서 다시 각 요소를 = 를 기준으로 나누면

이렇게 우리가 원하는 대로 예쁘게 나뉘어진다.
각 배열의 인덱스값을 이용해 데이터를 변수에 담고
해당하는 인풋태그(content1, content2, content3)의 value 값을 data로(hela, thor, loki)바꿔주겠다- 하는 코드다.

쓰려면 쓸 수야 있겠지만 이렇게 하면 다시 로드할 때 한글이나 특수문자 등은 전부 깨져서 나온다.
decodeURIComponent(); 를 따로 사용해줘야 한다.
그리고 계속 split 을 하기도 번거롭고.

dictionary

사전형 데이터를 만들어 저장하는 방법이다.
보통은 이 방법이 더 많이 쓰인다.

사전형이라고는 하지만 그냥 객체를 만드는 것이다.
오늘 객체에 대해 자세히 배운 것은 아니고… 프로퍼티와 값을 할당하고 불러오는 정도만 배웠다.
사전형 데이터를 저장하려면 JSON.stringify(); 를 써서 문자형 데이터로 바꿔서 저장해줘야 한다.
안 그러면 [object Object] 라고 나와서 마음이 아파진다.
저장한 문자형 데이터를 다시 사전형 데이터로 불러올 때는 JSON.parse(); 를 사용한다.

JSON.parse(); 를 하면 다시 쌍따옴표에 들어가지 않은 형태인
{name: "rudy"}
가 된다.

Validation, 유효성 검사

March 18, 2019 Javascript

유효성 검사란…

입력하는 값이 정해진 포맷에 맞는 것인지 확인하는 것이다.
사용자 입장에서는 회원가입할 때 자꾸 뭐가 덜 됐다며 빠꾸(…)를 먹이는 그것이다.
그렇다고 만드는 사람은 편하냐, 당연히 아니다…

오늘은 회원가입 창을 만들어 빈 값이 있는지 없는지를 확인하는 유효성 검사와
정규표현식을 이용한 유효성 검사를 해보았다.

빈 값인지 아닌지 검사하는 것은 간단하다.
submit 이벤트가 발생했을 때
문서에서 id 라는 아이디값을 가진 input 요소의 value(내가 입력한 id) 를 변수 id 에 담고
그 값이 비어있는지 조건문을 통해 확인한 뒤
만약 비어있다면 alert 창을 띄우고 return false 를 해주면 더 이상 진행되지 않고 그대로 머물러 있게 된다.
똑같은 방법으로 다른 요소들도 검사를 해주면 된다.

정규표현식을 이용한 검사는 이보다 조금 더 복잡하다.
단순히 빈 값이냐를 보는 것이 아니라
문자열이 어떤 형태나 패턴으로 되어있는지를 검사하는 것이다.
아이디는 영문자와 숫자를 혼합한 몇 자 이상이어야 한다… 이런 조건들 말이다.
정규표현식을 쓸 때 미리 테스트 해볼 수 있는 좋은 사이트다.
-> RegExr

변수에 정규표현식을 담아서 사용하는게 훨-씬 편하다.
복잡해보이는 코드가 자꾸 보이면 성가시다.
.test() 를 정규표현식 뒤에 붙여서 검사한다. 괄호 안에는 내가 검사할 요소를 집어넣는다.
내가 적은 id 가 정규표현식을 만족하는 것이 아니라면(느낌표가 붙었으니까)
alert 창을 띄우고 진행을 막는다.
