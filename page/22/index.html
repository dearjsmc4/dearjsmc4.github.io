float 이슈 해결해보기

April 1, 2019 HTML/CSS


이 기괴한 레이아웃을


이렇게 바꾸기.

요소를 정렬할 때는 flex 를 많이 쓴다고 하는데.
만약 익스플로러 하위버전까지 호환되어야 한다면 플렉스는 저멀리 안녕 하고 보내주어야 한다.
그리고 나에게는 float 나 inline 과 같은 선택지가 주어지는데,
인라인같은 경우는 요소들 사이에 공백문자가 생겨버려서 이 상황엔 적절치 않고
float 를 쓰면 정렬할 수 있다.
그런데 float 는 말 그대로 다른 요소들 위에 붕 떠있게 되기 때문에
생각 없이 쓰다보면 레이아웃이 다 망가지게 된다.

현재 박스1과 박스2에 float:left 를 준 상태이다.
float 로 정렬까지는 했는데 이 망가진 레이아웃을 어떻게 고칠 수 있을까.

1. clear: both;


박스 4에 clear:both 를 주게 되면 박스 4는 float 로부터 벗어나 그 아래에 잘 놓이게 되지만,
박스 1, 박스2 의 부모요소는 여전히 그 자식요소의 크기를 놓치고 있다.

2. overflow


박스1,2의 부모요소 에 overflow:visible 을 줬을 때.
visible 값을 주게 되면 같은 문제가 생긴다.
visible 값을 사용해서 float 이슈를 해결하려면
부모 요소에 너비와 크기를 명확히 지정해줘야 한다.
그런데 나는 박스1,2의 크기만큼 부모가 유동적으로 바뀌기를 원하기 때문에 높이값을 지정하고싶지 않다.
그럼 hidden 값을 주면?


놀랍게도 문제가 해결되었다!
내가 원하는 모양대로 잘 나와주었다.

​
예시를 위해 박스 2의 스타일을
position: relative;
left: 400px;
정도(다분히 고의적으로) 주게 되면부모요소에 주어진 overflow:hidden 속성에 충실하게도
내 박스 2가 제대로 보이지 않게 된다.
지금은 괜찮지만 나중에 문제가 생길 수도 있다.

3. CSS 가상요소

스타일 시트에 위와 같은 코드를 작성한다.

div3 은 박스1,2의 부모요소(노란색박스)의 아이디다.
::after 는 div3의 마지막에 가상의 자식노드를 생성한다.
( 반대로 ::before 는 가장 처음에 가상의 자식노드를 생성한다.)
이렇게 만들어진 가상 요소를 float 해결을 위해 쓰려면 반드시 content 가 있어야 하는데,
만약 content 에 값을 주게 되면,

​
박스1과 2 아래에 “나는 가상으로 만들어진 노드!” 라는 content 텍스트가 보이게 된다.
나는 이 가상요소에 어떤 텍스트도 보이지 않기를 원하기에 그냥 “” 로 비워두기로 했다.

그리고 이 가상요소는 날 때부터 인라인 요소인데
clear:both 는 인라인 요소에는 적용되지 않는다. 인라인블록에도 해봤는데 먹히지 않는다.
그래서 이 가상노드의 display 값을 block 으로 설정해주어서
clear:both 가 적용되도록 한다.

​
오버플로우로 인한 문제도 해결되었다.
가상요소를 사용하면 overflow:hidden 을 썼을 때 박스가 잘려보이던 문제도 해결이 된다.

박스사이징, content-box & border-box

April 1, 2019 HTML/CSS

오늘 CSS 박스모델을 배우면서 콘텐트박스와 보더박스에 대해 배웠는데
헷갈리는 부분이 있어서 정리해보기로 했다.

box-sizing

box-sizing 속성은 박스의 크기를 결정하는 방식을 설정한다.
내가 임의로 div 박스 두 개를 만들어보았고 오른쪽은 각 박스의 스타일 속성이다.
구분을 쉽게 하기 위해 너비를 주로 볼 것이다.
마진값은 얼마를 주든 투명하니까, 일단 보여지는 너비를 가지고 얘기하자.

​

박스 두 개의 width 값은 300px 로 같은데, 왜 보여지는 크기는 다른 것일까?
박스사이징 속성이 다르기 때문이다.
박스사이징으로 달라진 결과는 다음과 같다.

​

div1은 box-sizing: content-box​ 로 설정되었다.
컨텐츠부분의 너비는 300px로 고정이 되고 그 후에 패딩값과 테두리값이 더해졌다.
그래서 보여지는 총 너비, 즉 테두리까지 포함한 너비는
300px + 100px(양쪽 패딩값) + 10px(양쪽 테두리값) = 410px이 된다.

​
​​

div2는 box-sizing: border-box로 설정되었다.
그러자 컨텐츠부분의 너비가 300px이 아닌 190px 로 줄어든 것을 확인할 수 있다.
그래서 보여지는 총 너비는 190px + 100px + 10px = 300px 이 되고,
내가 원래 설정한 width 값인 300px에 꼭 맞게 된다.

content-box

content-box​ 는 컨텐츠 너비, 패딩, 테두리까지 전부 너비에 포함하기 때문에
내가 width 를 300px 로 설정했더라도
패딩이나 테두리값을 주면 그 크기가 너비에 포함되어 더 커지게 된다.
그리고 내가 실제로 사용할 수 있는 컨텐츠부분의 너비가 그대로 300px 로 고정될 수 있다.

border-box

border-box 는 내가 정한 width 값을 벗어나지 않게 해준다.
박스 안에 여백을 얼마나 주든 테두리를 얼마나 주든,
전체 너비는 내가 지정한 300px 그대로 고정된다.
대신, 안쪽 여백을 주게 되면 내가 실제로 사용할 수 있는 부분이 그만큼 줄어들게 된다.

정리

간단하게 기억하려면 이름 그대로 생각하면 될 것 같다.

content-box 는 내가 실제로 사용할 수 있는 컨텐츠 너비를 기준으로 크기를 정하는 것,
(내가 설정한 width 가 곧 내가 실제로 사용할 수 있는 너비가 되니까)

border-box 는 가장 바깥쪽 테두리를 기준으로 크기를 정하는 것.
(내가 설정한 width 가 박스 전체의 크기가 되니까)
