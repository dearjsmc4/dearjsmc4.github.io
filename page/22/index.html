Image Replacement 기법

April 11, 2019 HTML/CSS

IR 기법

버튼의 텍스트를 마크업 상에서 그대로 유지하면서 보이지는 않게 하는 법.
기법이라고는 하지만 엄청나게 대단하거나 복잡하진 않다.
대부분 내가 아는 속성들을 응용했을 뿐이었다.

버튼 만들기

우선 버튼부분의 마크업은 아래와 같고,
CSS의 박스사이징은 border-box 값을 준 상태다. 중요

아무런 스타일도 주지 않은 상태이고, 버튼의 내용을 텍스트로 적어주었다.
이렇게 하면 화면낭독기가 버튼의 내용을 읽어줄 수 있고
사용자가 버튼의 사용목적을 명확히 알 수 있을 것이다.
이제 버튼의 크기를 설정해주자.

버튼의 크기가 작아지자 텍스트가 오버플로우되는 상황이 발생하였다.
여기서 중요한 트릭이 나온다.

padding-top

버튼 안에 padding-top 을 주어서 텍스트를 밖으로 밀어내는 것이다.
위쪽에서 박스사이징을 border-box 로 주는 것이 중요하다고 했던 이유가 바로 이것이다.
border-box는 패딩값을 추가로 준다고 해서 버튼의 크기가 늘어나지 않기 때문이다.
(border-box 와 content-box에 대해 포스팅한 적이 있으니 참고)

각 버튼에 padding-top:22px 을 주어서 텍스트를 밀어낸 결과.
보더박스는 박스 크기에 변함이 없는 반면, 콘텐트박스는 패딩값만큼 박스높이가 늘어나버렸다.
의도대로 잘 나온 보더박스 쪽으로 진행을 계속 해야 한다.
버튼에 넣고자 하는 아이콘을 배치해준다.

overflow

이제 텍스트를 숨겨야 하니
버튼에 overflow:hidden; 을 사용해서 텍스트를 숨겨준다.
( display:none; 이나 visibility:hidden; 을 사용하게 되면
화면 낭독기가 텍스트를 읽지 않고 지나가게 된다고 하니 주의하자.
웹 접근성을 위해 이런 복잡한 길로 가고 있는 것인데 말짱 도루묵이 되어버리면 안 되니 말이다.)

정의형 리스트, Definition List

April 8, 2019 HTML/CSS

정의형 리스트

li 태그와 h1~h6 태그와 p 태그 등을 번거롭게 쓰는 것 대신
정의형 리스트를 활용해보자.

  • dl : definition list, 용어를 정의하는 목록을 만든다.
  • dt : definition term, 정의하고자 하는 용어의 제목을 넣는다.
  • dd : definition description, 용어의 설명을 넣는다.

dt와 dd는 반드시 일대일로 대응되어야 하는 것은 아니다.
하나의 dt 아래에 여러 개의 dd 가 올 수도 있다.
그리고 HTML 5.2 부터는 dl 안에 div 를 넣어서 dt 와 dd 를 감쌀 수 있다.

주의사항

HTML 5.2: 4.4 Grouping content

Each term-description group consists of one or more terms (represented by dt elements) possibly as children of a div element child, and one or more descriptions (represented by dd elements possibly as children of a div element child), ignoring any nodes other than dt and dd element children, and dt and dd elements that are children of div element children within a single dl element.

  1. 각 정의형 리스트는 div 요소 내에 한 개 이상의 dt, dd 를 가질 수 있다.
  2. dl 의 div 안에는 dt 나 dd 가 아닌 요소를 넣을 수 없다.

그리고

  1. 단일 dl 요소 내에 dt, dd 를 묶은 div 요소와 dt, dd 요소는 서로 형제로서 존재할 수 없다.

위의 코드를 HTML Validator 로 돌려보면 에러 표시가 난다.
div 요소 밖에 dd 와 dt 가 존재하면서 둘 사이가 형제요소가 되기 때문이다.
dl 은,

  1. 한 개 이상의 dd 가 따라붙는 한 개 이상의 dt 요소들로 구성되거나
  2. 한 개 이상의 div 요소로 구성될 수 있다.

둘 중 하나의 상황만 허용된다는 것이다.

위와 같은 코드는 오류가 나지 않는다.
dt 와 dd 가 따로 떨어져있는게 아니라 div 요소 안에 자식요소로서 존재하기 때문이다.
