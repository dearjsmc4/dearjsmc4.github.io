정의형 리스트, Definition List

April 8, 2019 HTML/CSS

정의형 리스트

li 태그와 h1~h6 태그와 p 태그 등을 번거롭게 쓰는 것 대신
정의형 리스트를 활용해보자.

  • dl : definition list, 용어를 정의하는 목록을 만든다.
  • dt : definition term, 정의하고자 하는 용어의 제목을 넣는다.
  • dd : definition description, 용어의 설명을 넣는다.

dt와 dd는 반드시 일대일로 대응되어야 하는 것은 아니다.
하나의 dt 아래에 여러 개의 dd 가 올 수도 있다.
그리고 HTML 5.2 부터는 dl 안에 div 를 넣어서 dt 와 dd 를 감쌀 수 있다.

주의사항

HTML 5.2: 4.4 Grouping content

Each term-description group consists of one or more terms (represented by dt elements) possibly as children of a div element child, and one or more descriptions (represented by dd elements possibly as children of a div element child), ignoring any nodes other than dt and dd element children, and dt and dd elements that are children of div element children within a single dl element.

  1. 각 정의형 리스트는 div 요소 내에 한 개 이상의 dt, dd 를 가질 수 있다.
  2. dl 의 div 안에는 dt 나 dd 가 아닌 요소를 넣을 수 없다.

그리고

  1. 단일 dl 요소 내에 dt, dd 를 묶은 div 요소와 dt, dd 요소는 서로 형제로서 존재할 수 없다.

위의 코드를 HTML Validator 로 돌려보면 에러 표시가 난다.
div 요소 밖에 dd 와 dt 가 존재하면서 둘 사이가 형제요소가 되기 때문이다.
dl 은,

  1. 한 개 이상의 dd 가 따라붙는 한 개 이상의 dt 요소들로 구성되거나
  2. 한 개 이상의 div 요소로 구성될 수 있다.

둘 중 하나의 상황만 허용된다는 것이다.

위와 같은 코드는 오류가 나지 않는다.
dt 와 dd 가 따로 떨어져있는게 아니라 div 요소 안에 자식요소로서 존재하기 때문이다.

박스사이징, content-box & border-box

April 1, 2019 HTML/CSS

오늘 CSS 박스모델을 배우면서 콘텐트박스와 보더박스에 대해 배웠는데
헷갈리는 부분이 있어서 정리해보기로 했다.

box-sizing

box-sizing 속성은 박스의 크기를 결정하는 방식을 설정한다.
내가 임의로 div 박스 두 개를 만들어보았고 오른쪽은 각 박스의 스타일 속성이다.
구분을 쉽게 하기 위해 너비를 주로 볼 것이다.
마진값은 얼마를 주든 투명하니까, 일단 보여지는 너비를 가지고 얘기하자.

​

박스 두 개의 width 값은 300px 로 같은데, 왜 보여지는 크기는 다른 것일까?
박스사이징 속성이 다르기 때문이다.
박스사이징으로 달라진 결과는 다음과 같다.

​

div1은 box-sizing: content-box​ 로 설정되었다.
컨텐츠부분의 너비는 300px로 고정이 되고 그 후에 패딩값과 테두리값이 더해졌다.
그래서 보여지는 총 너비, 즉 테두리까지 포함한 너비는
300px + 100px(양쪽 패딩값) + 10px(양쪽 테두리값) = 410px이 된다.

​
​​

div2는 box-sizing: border-box로 설정되었다.
그러자 컨텐츠부분의 너비가 300px이 아닌 190px 로 줄어든 것을 확인할 수 있다.
그래서 보여지는 총 너비는 190px + 100px + 10px = 300px 이 되고,
내가 원래 설정한 width 값인 300px에 꼭 맞게 된다.

content-box

content-box​ 는 컨텐츠 너비, 패딩, 테두리까지 전부 너비에 포함하기 때문에
내가 width 를 300px 로 설정했더라도
패딩이나 테두리값을 주면 그 크기가 너비에 포함되어 더 커지게 된다.
그리고 내가 실제로 사용할 수 있는 컨텐츠부분의 너비가 그대로 300px 로 고정될 수 있다.

border-box

border-box 는 내가 정한 width 값을 벗어나지 않게 해준다.
박스 안에 여백을 얼마나 주든 테두리를 얼마나 주든,
전체 너비는 내가 지정한 300px 그대로 고정된다.
대신, 안쪽 여백을 주게 되면 내가 실제로 사용할 수 있는 부분이 그만큼 줄어들게 된다.

정리

간단하게 기억하려면 이름 그대로 생각하면 될 것 같다.

content-box 는 내가 실제로 사용할 수 있는 컨텐츠 너비를 기준으로 크기를 정하는 것,
(내가 설정한 width 가 곧 내가 실제로 사용할 수 있는 너비가 되니까)

border-box 는 가장 바깥쪽 테두리를 기준으로 크기를 정하는 것.
(내가 설정한 width 가 박스 전체의 크기가 되니까)
