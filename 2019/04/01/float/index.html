float 이슈 해결해보기

April 1, 2019 HTML/CSS


이 기괴한 레이아웃을


이렇게 바꾸기.

요소를 정렬할 때는 flex 를 많이 쓴다고 하는데.
만약 익스플로러 하위버전까지 호환되어야 한다면 플렉스는 저멀리 안녕 하고 보내주어야 한다.
그리고 나에게는 float 나 inline 과 같은 선택지가 주어지는데,
인라인같은 경우는 요소들 사이에 공백문자가 생겨버려서 이 상황엔 적절치 않고
float 를 쓰면 정렬할 수 있다.
그런데 float 는 말 그대로 다른 요소들 위에 붕 떠있게 되기 때문에
생각 없이 쓰다보면 레이아웃이 다 망가지게 된다.

현재 박스1과 박스2에 float:left 를 준 상태이다.
float 로 정렬까지는 했는데 이 망가진 레이아웃을 어떻게 고칠 수 있을까.

1. clear: both;


박스 4에 clear:both 를 주게 되면 박스 4는 float 로부터 벗어나 그 아래에 잘 놓이게 되지만,
박스 1, 박스2 의 부모요소는 여전히 그 자식요소의 크기를 놓치고 있다.

2. overflow


박스1,2의 부모요소 에 overflow:visible 을 줬을 때.
visible 값을 주게 되면 같은 문제가 생긴다.
visible 값을 사용해서 float 이슈를 해결하려면
부모 요소에 너비와 크기를 명확히 지정해줘야 한다.
그런데 나는 박스1,2의 크기만큼 부모가 유동적으로 바뀌기를 원하기 때문에 높이값을 지정하고싶지 않다.
그럼 hidden 값을 주면?


놀랍게도 문제가 해결되었다!
내가 원하는 모양대로 잘 나와주었다.

​
예시를 위해 박스 2의 스타일을
position: relative;
left: 400px;
정도(다분히 고의적으로) 주게 되면부모요소에 주어진 overflow:hidden 속성에 충실하게도
내 박스 2가 제대로 보이지 않게 된다.
지금은 괜찮지만 나중에 문제가 생길 수도 있다.

3. CSS 가상요소

스타일 시트에 위와 같은 코드를 작성한다.

div3 은 박스1,2의 부모요소(노란색박스)의 아이디다.
::after 는 div3의 마지막에 가상의 자식노드를 생성한다.
( 반대로 ::before 는 가장 처음에 가상의 자식노드를 생성한다.)
이렇게 만들어진 가상 요소를 float 해결을 위해 쓰려면 반드시 content 가 있어야 하는데,
만약 content 에 값을 주게 되면,

​
박스1과 2 아래에 “나는 가상으로 만들어진 노드!” 라는 content 텍스트가 보이게 된다.
나는 이 가상요소에 어떤 텍스트도 보이지 않기를 원하기에 그냥 “” 로 비워두기로 했다.

그리고 이 가상요소는 날 때부터 인라인 요소인데
clear:both 는 인라인 요소에는 적용되지 않는다. 인라인블록에도 해봤는데 먹히지 않는다.
그래서 이 가상노드의 display 값을 block 으로 설정해주어서
clear:both 가 적용되도록 한다.

​
오버플로우로 인한 문제도 해결되었다.
가상요소를 사용하면 overflow:hidden 을 썼을 때 박스가 잘려보이던 문제도 해결이 된다.
