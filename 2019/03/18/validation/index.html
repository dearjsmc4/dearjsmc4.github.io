Validation, 유효성 검사

March 18, 2019 Javascript

유효성 검사란…

입력하는 값이 정해진 포맷에 맞는 것인지 확인하는 것이다.
사용자 입장에서는 회원가입할 때 자꾸 뭐가 덜 됐다며 빠꾸(…)를 먹이는 그것이다.
그렇다고 만드는 사람은 편하냐, 당연히 아니다…

오늘은 회원가입 창을 만들어 빈 값이 있는지 없는지를 확인하는 유효성 검사와
정규표현식을 이용한 유효성 검사를 해보았다.

빈 값인지 아닌지 검사하는 것은 간단하다.
submit 이벤트가 발생했을 때
문서에서 id 라는 아이디값을 가진 input 요소의 value(내가 입력한 id) 를 변수 id 에 담고
그 값이 비어있는지 조건문을 통해 확인한 뒤
만약 비어있다면 alert 창을 띄우고 return false 를 해주면 더 이상 진행되지 않고 그대로 머물러 있게 된다.
똑같은 방법으로 다른 요소들도 검사를 해주면 된다.

정규표현식을 이용한 검사는 이보다 조금 더 복잡하다.
단순히 빈 값이냐를 보는 것이 아니라
문자열이 어떤 형태나 패턴으로 되어있는지를 검사하는 것이다.
아이디는 영문자와 숫자를 혼합한 몇 자 이상이어야 한다… 이런 조건들 말이다.
정규표현식을 쓸 때 미리 테스트 해볼 수 있는 좋은 사이트다.
-> RegExr

변수에 정규표현식을 담아서 사용하는게 훨-씬 편하다.
복잡해보이는 코드가 자꾸 보이면 성가시다.
.test() 를 정규표현식 뒤에 붙여서 검사한다. 괄호 안에는 내가 검사할 요소를 집어넣는다.
내가 적은 id 가 정규표현식을 만족하는 것이 아니라면(느낌표가 붙었으니까)
alert 창을 띄우고 진행을 막는다.
